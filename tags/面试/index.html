如何停止一个线程？

  • stop方法被废弃,为什么不能简单的停止一个线程，线程直接停止涉及到资源清理问题，会造成内存数据的异常
  • Interrupt中断
    • Interrupted()是静态方法，获取当前线程的中断状态，并清空该状态,如果是状态状态调用后会返回false
    • isInterrupted()是非静态方法，获取当前线程的中断状态，不清空状态，可以重复调用，中断请空前一直返回true
    • boolean标志位 ，注意线程可见性问题，要加volatile

线程安全

  • 什么是线程安全
    • 可变资源（内存）线程间共享
  • 怎么实现线程安全
    • 不共享资源
      • ThreadLocal（本质上是ThreadLocalMap）
        • 声明为全局静态final成员
        • 避免储存大量对象
        • 用完后及时移除对象
    • 共享不可变资源
      • 通过final实现（扩展final的作用，例如禁止重排序）
      • 禁止重排序 volatile
    • 共享可变资源
      • 可见性
        • 使用final关键字
        • 使用volatile关键字
        • 加锁，锁释放时会强制将值刷新到主内存
      • 操作原子性
        • 加锁，保证操作的互斥性
        • 使用CAS指令（如Unsafe.compareAndSwapInt）
        • 使用原子数值类型（如AtomicaInteger）
        • 使用原子属性更新器（AtomicReferenceFieldUpdater）
      • 禁止重排序

如何进行锁优化

  • 长锁不如短锁：尽可能只锁必要的部分

AtomicReference和AtomicReferenceFiledUpadter

  • AR和ARFU功能一致，原理相同，都基于Unsafe的CAS操作
  • AR通常作为对象的成员使用，占16B（指针压缩）、24B (指针不压缩)
  • ARFU通常作为类的静态成员使用，对实例成员进行修改
  • AR使用更友好，ARFU更适合类实例比较多的场景

面试宝典

1.在浏览器地址栏输入URL，按下回车后的处理过程

  • 浏览器会把输入的域名解析成对应的IP,过程如下：
    1. 查找浏览器缓存，浏览器一般会缓存DNS一段时间，如果有该域名对应的缓存则直接返回
    2. 查找系统缓存，如果浏览器中找不到缓存，浏览器会进行系统调用，查找本机host文件，如果找到则直接返回对应的IP，没有则进行下一步
    3. 查找路由器缓存，如果前两步都没有找到对应IP，会将请求发送给路由器，查找ISP服务商缓存DNS的服务器
    4. 递归查询，若以上步骤找不到，ISP服务商会向其他根域名服务器继续请求对应的IP地址，若是根域名服务器找不到会继续向上一层级的域名服务器查找
    5. 迭代查询，本地域名服务器采用迭代查询，先向一个根域名服务器查询，当根域名服务器收到查询请求后，要么返回IP地址，要么告诉本地域名服务器下一步该查询哪个服务器，交给本地域名服务器进行后续查找

  • 浏览器与目标服务器建立TCP连接

    浏览器通过DNS解析得到了服务器的IP地址之后，与服务器建立TCP连接，TCP三次握手，浏览器所在的客户端向服务器发出连接请求报文（SYN标志为1）;服务器收到报文后，同意建立连接，向客户端发出确认报文（SYN,ACK标志位均为1）；客户端收到报文后，再次向服务器发出报文，确认已接收到服务器发出的确认报文。此时客户端与服务器之间的TCP连接建立成功。

  • 浏览器通过http/https协议方发送请求
    浏览器向服务器发起一个HTTP_GET请求。请求中包含访问的URL、HEADERS等

  • 重定向
    对于大型网站存在多个服务器，为了负载均衡或导入流量，往往不会直接返回请求页面，而是重定向。返回的状态码就不是200，而是301或者302的重定向码，浏览器接收到重定向相应后，会重新访问服务器返回的重定向地址

  • 服务器返回HTML响应
    服务器接收到浏览器的GET请求后，若是不需要重定向并且请求参数正确，会返回相应的HTML网页内容，此时的状态为200。

  • 浏览器渲染页面
    浏览器接收到服务器返回的网页内容后，根据浏览器的渲染机制对相应的数据进行渲染。渲染后的数据进行相应的页面呈现和脚本的交互

  • 释放TCP连接

    1. 浏览器所在客户端向服务器发出释放连接报文，停止发送数据
    2. 服务器接受到释放请求报文后，发出确认报文，然后继续将未传送的数据发送完
    3. 服务器发送完成数据后，向客户端发送释放报文，表示可以释放连接了
    4. 客户端接收到报文后，发出确认报文，确认已经收到消息，并在等待一段时间后释放TCP连接

2.TCP协议

传输控制协议（TCP,Transmission Control Protocol ）是一种面向连接的、可靠的、基于字节流的的传输层控制协议。

  • 应用层向TCP层发送用于网间传输的、用8位字节表示的数据流，TCP把数据流分割成适当长度的报文段，之后TCP把数据包传给IP层，由IP层通过网络将数据传输给接收端的TCP层。

  • TCP层保证了报文传输的可靠，传输时，会给每个包一个序号，序号保证了接收端的按序接受。接收端对成功接受的包会发回一个相应的确认（ACK）;如果发送端在合理的往返时延（RTT）内未接收到确认信息，就认为该段数据丢失，进行重新传输。

  • TCP协议保证了数据的正确性和合法性，TCP用一个校验函数在发送和接受数据的时候进行计算校验，同时也可以使用MD5对数据进行加密

  • 在流量控制上，采用滑动窗口协议，协议中规定，对于窗口内未经确认的分组需要重传

TCP的特点：

- 基于流的方式
- 面向连接
- 数据有序且可靠
